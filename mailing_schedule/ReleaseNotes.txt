Програма для розсилки завдань студентам та збирання ярезультатів

Версія 2.0

Зміни та доповнення у порівнянні з попередньою версією
1. З 30.05.22 Google припинив підтримку входу у аккаунт для «небезпечних» застосувань. Для продовження використання програми треба зробити декілька кроків:
	a.	Задати двохфакторну аутентифікацію для свого аккаунта, з якого програма розсилає завдання та на який приймає відповіді від студентів. Як це	 зробити, можна прочитати тут: https://support.google.com/accounts/answer/185839?hl=ru&co=GENIE.Platform%3DDesktop
	b.	Створити пароль для свого застосування (програми). Пароль може бути єдиним для усіх програм у Python, що запускаються з даного комп’ютера. Як це зробити, описано тут: https://support.google.com/accounts/answer/185833?hl=ru. З списку видів застосувань треба вибрати «інше», ввести довільну назву. У відповідь буде сформований пароль з 16 символів. Цей пароль треба вставити у файл config.txt у каталозі з програмою замість Вашого паролю для входу у аккаунт пошти.
2. Розділено варіанти програми для написання модульного контролю та іспиту. Для варіанту з іспитом передбачено окрему розсилку теоретичних та практичних завдань з іспиту. Задається час окремо для розсилки теоретичного та практичного завдань. Збирання результатів відбувається як і раніше. Варіант для модульного контролю – у папці mod_cntrl, для іспиту – у папці exam
3. Змінилась структура папки з файлами (більш докладно у відповідній папці (mod_cntrl або exam)  у файлі Readme.txt)
4. Виправлено помилку розсилки завдань для великих потоків (100 + студентів). Розсилка здійснюється частинами (chunks) та повторюється для окремого адресату, якщо спроба не вдалась. Здійснюється до 3 спроб.
5. Підвищено стійкість програми у рекції на помилки при отриманні листів з різних поштових серверів.
6. Додано реєстрацію студентів на модульну контрольну або іспит. Для реєстрації студенту треба написати листа довільного змісту у визначений день та час на адресу викладача (насправді перевіряється тільки день написання листа). Усі, хто не написав листа, вважаються такими, хто не з'явився на модульну контрольну (залік, іспит) і їм не відправляються завдання. Програма аналізує файл group.txt зі списком групи студентів та будує файл отримувачів завдань recipients.txt з тих студентів, хто зареєструвався. Більш докладно це описано у файлі Readme.txt
